# 项目通用规则 (General Project Rules) 
本项目为 Tauri 2.0 + WebView2 桌面端应用，包含深度系统交互。所有规则为强制执行。

## 1. 代码与开发规范 
- **缩进规范**：**必须**统一使用 2 个空格。 
- **注释规范**：编写新函数**必须**附带 JSDoc 或 Docstring 注释。 
- **模块化**：**必须**编写可测试的模块化代码，遵守 DRY 和单一职责原则。

## 2. 全栈技术规范 
- **前端**：布局**必须**采用 Flexbox 或 CSS Grid，**严禁**硬编码像素值；组件**必须**保持单一职责。
- **后端 API**：路由**必须**遵循 RESTful 规范（如 `/api/v1/user-profiles`）；返回**必须**为标准 JSON 并使用正确 HTTP 状态码。
- **数据库**：**绝对禁止**直接拼接 SQL，**必须**使用参数化查询或 ORM；结构变更**必须**通过 Migrations 脚本记录。

## 3. 系统级桌面多窗口与截图开发宪法 (Tauri 2.0 & WebView2) 
* **技术选型决策矩阵**：涉及深度交互项目，Agent **必须**以表格列出至少两种备选选型对比，由用户选择后确定。 
* **窗口防卡死死锁**：透明辅助窗口**必须**在 `setup` 期间主线程静默预加载并 `visible(false)`。运行时仅执行 `show()` / `hide()`。 
* **主线程调度律**：操纵窗口句柄、位置与尺寸的 API，**必须强制**使用 `app_handle.run_on_main_thread(move || { ... })` 派发。 
* **多显示器 DPI 换算**：多屏截图**必须**遍历 `available_monitors` 注入物理原点和缩放比。前端画框后折算绝对物理像素坐标传回 Rust，**严禁**在 Rust 端混淆多屏坐标系。 
* **防进程残留**：关闭 UI **必须**实现为 `app_handle.exit(0)`，并监听主窗口 `Destroyed` 事件强制退出，**严禁**残留进程锁定 WebView2 目录。 

## 4. Git 提交规范 
- **绝对禁止**在用户未自测通过或未下达明确指令前，自行执行 `git commit` 或 `push`。 
- Commit Message **必须**遵循前缀规范：`feat:` / `fix:` / `docs:` / `style:` / `refactor:` / `test:`

## 5. 额度管控与执行策略 (Token & Action Control)
- **前置澄清律**：在处理涉及多文件修改或架构变更的复杂需求时，**绝对禁止**直接开始写代码。你必须先输出一个极简的“实施计划”（不超过 5 个要点），并向我确认：“是否按此计划执行？”
- **最小可行修改**：修复 Bug 或新增功能时，**必须**遵循最小改动原则。严禁为了“代码美观”顺手重构无关模块，这会极大消耗上下文额度。
- **拒绝过度设计**：不要为未发生的需求预留接口。实现当前需求即可，保持代码最简形态。

## 6. Token 极简输出宪法
- **上下文极简原则**：回答问题时，不要重复我的问题，不要输出客套话。直接给出结论或代码。
- **禁止复读代码**：修改代码时，**仅输出修改的函数或 Diff 片段**，严禁输出未修改的代码块。
- **工具调用克制**：如果已经知道某个文件的路径，直接读取该文件，不要先调用 `list_directory` 再逐级查找，减少工具调用次数。
- **单次任务闭环**：尽量在一次回复中完成代码编写，避免“写一半让你自己看看”这种拖沓行为。
